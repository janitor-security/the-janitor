    2026-04-09 — GA Teardown Audit (Hostile Product Review)

    Verdict: not a toy engine; still not frictionless GA product for a 10,000-developer org.

    Already present — do not waste cycles re-building these: - SARIF output exists (janitor report --format sarif) and GitHub SARIF upload wiring already exists. - CLI progress spinners already exist for scan / clean flows; the gap is long-run orchestration and resumability, not absence of all progress UI. - Suppression primitives exist (# noqa-style pragmas, test-scope routing, local-brain janitor pardon); the gap is governable, reviewable suppression lifecycle.

    P2 — Scale / Platform Expansion

    P2-1 — No first-class drift dashboard for policy health

    Gap: The ledger is strong after the fact. There is still no compact view showing suppression growth, noisy rules, top owners, stale waivers, or score drift across services.

    Required change: 1. Add a policy-health dashboard and JSON endpoint. 2. Track waiver counts, expired waivers, and hottest findings per team.

    P2-2 — No lightweight maintainer mode for tiny repos

    Gap: Free capability is generous, but the product packaging still feels enterprise-first. Small OSS maintainers need a "minimal noise" preset with lower ceremony, tiny config, and human-readable output defaults.

    Required change: 1. Add janitor init --profile oss. 2. Add compact text summaries optimized for solo maintainers. 3. Publish migration guidance from zero config to governed config.
