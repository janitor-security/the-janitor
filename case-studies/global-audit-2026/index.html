    What Changed in v6.12.7

    Two engine corrections materially changed the precision of this audit compared to previous runs:

    Agnostic IaC Shield Bypass: Files with extensions .nix, .lock, .json, .toml, .yaml, .yml, .csv bypass ByteLatticeAnalyzer before entropy analysis runs. In NixOS/nixpkgs, nix sha256 hashes (base64 encoded, ~6 bits/char) previously triggered AnomalousBlob detections — the IaC Shield eliminates these false positives entirely.

    Zombie Dep Scanner Gating: find_zombie_deps_in_blobs is now gated on registry_loaded — the scanner only runs when a full .janitor/symbols.rkyv registry is available. Without a registry, a PR that bumps a manifest but doesn't touch source files would have all deps appear as zombies (diff-only false positives). The v6.12.7 fix produces only true-positive zombie dep findings.

    Result: The 124 antipatterns in this audit are confirmed structural defects — no entropy noise, no manifest false positives.

    Forensic Spotlights

    Spotlight 1: rust-lang/rust — 24 Vacuous unsafe Blocks

    Result: 100/100 PRs bounced. 24 confirmed antipatterns. 100% unlinked to issues.

    The language whose entire value proposition is memory safety submitted PRs containing vacuous unsafe blocks — unsafe invoked for code that needs no such verification. No raw pointer dereferences. No FFI calls. No inline assembly.

    Worst offender: PR #153239 by asder8215 — score 1,235, 8 vacuous unsafe blocks:

    Antipatterns: Vacuous unsafe block: contains no raw pointer dereferences,
              FFI calls, or inline assembly (×8)
No linked issue.

    This is not a minor style issue. Every unsafe block is a contract. When that contract is invoked for code that requires no verification, it trains reviewers to ignore unsafe as a signal and inflates the soundness audit surface.

    Spotlight 2: apache/kafka — Java Debug Logging at Scale

    Result: 100/100 PRs bounced. 16 antipatterns. 100% unlinked to issues.

    apache/kafka produced the highest Java antipattern count in the corpus. PR #21580 by aliehsaeedii — System.out.println debug logging committed to production Kafka source:

    Score: 800 | Antipattern: System.out.println: console debug logging in production
           | — use a structured logger (SLF4J, Log4j, etc.)

    System.out is synchronised on the PrintStream monitor, is not configurable by log level, and writes to stdout with no structured metadata. In any production Java codebase the invariant is: use SLF4J or Log4j. System.out.println in committed code is debug logging that was never cleaned up.

    Spotlight 3: NixOS/nixpkgs — IaC Shield Validated

    Result: 100/100 PRs bounced. 0 antipatterns. IaC Shield bypass confirmed working.

    In previous engine versions, Nix sha256 hashes (base64-encoded, ~6 bits/char) triggered AnomalousBlob detections — false positives that scored automation PRs as if they contained binary payloads. The v6.12.7 Agnostic IaC Shield bypass specifically exempts .nix, .lock, .json, .toml, .yaml, .yml, and .csv files from ByteLatticeAnalyzer.

    Result: Every r-ryantm automation PR scores exactly 70 (unlinked-only) — no more entropy noise.

    The Precision Guarantee

    The 124 antipatterns in this audit are not estimates. They are confirmed structural defects detected by tree-sitter AST walks against 2,090 live pull requests from 22 production codebases. The v6.12.7 engine corrections (IaC Shield, registry-gated zombie deps) have eliminated the two most common sources of false positives from previous runs.

    The Universal Bot Shield

    The is_automation_account() check runs a 4-layer classification before any bounce analysis: 1. app/ prefix — GitHub Apps API format (app/dependabot, app/renovate, app/copilot-swe-agent) 2. [bot] suffix — legacy bot naming convention 3. trusted_bot_authors — configurable allowlist in janitor.toml 4. [forge].automation_accounts — per-repo custom bot accounts

    Every bot PR still receives full structural analysis — dead symbols, clones, antipatterns. Bots are not exempt from review; they are correctly classified for reporting purposes.
